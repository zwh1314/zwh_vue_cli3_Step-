青马班选课系统——学生端前端（Step Ⅰ）
1. 选课模块（Select_course.vue)：根据后台数据库提供给前端的课程对象数组，可在该模块显示相应课程，相关课程属性可根
据以后需求进行修改，在本版本中，我列举了几门课程展示效果，点击选课按钮，即可将相应课程即可传递给后端并加入到学生
相关数据库。并且点击课程可以获得相应的【特定】的课程信息。

2.课表模块：分为已修课程(Course_Table_Already.vue)和待修课程(Course_Table_Ready.vue)。已修课程页面提供评课页面，
点击评课按钮，即可对相应课程进行评教（三项内容为必填，任意一项或多项不填无法成功评教），并上传数据库。评教成功，
其评教状态也会相应改变。待修课程提供退课按钮，点击该按钮，即可将课程移除课程表，通知后端删除数据库相应课程即可，
点击相应课程也可查看课程特点内容。

3. 成绩单（Grade_table.vue)：分为必修学分(Must_course_table.vue)和选修学分(Select_course_table.vue)和失信记录
(Lose_trust.vue)页面。其中各页面均可从后端数据库获取数据并进行显示，三个页面可通过成绩单页面点击相应链接进入。

4. 签到页面和提示信息页面，有一定构思，缺乏后端，不好进行，故未加入此版本。（在加入后端后，在每个用户登陆成功后，
可为每个用户开两个线程，专门去检测（监听）签到和各个提示信息产生条件，并在各条件发生时，控制前端产生alert弹框即可。
加入弹框部分较容易，且需与后端交接，故计划在下一步开发中实现。

5. 个人信息（Student_inf.vue）：简单制作了一个个人信息表，可根据需求进行相应修改，在用户登陆成功后从后端数据库获
取用户数据后填入即可。

                             2. npm run serve
